一、http状态码

服务器对客户端（如浏览器、APP）请求的“响应状态标识”，由三位数字组成，首位数字定义了响应的核心类别，共分为5大类，不同状态码对应不同的请求处理结果，是前端调试、后端排错和网络通信的核心“语言”

1、 1xx 信息响应

服务器已接收请求，需客户端继续操作，例如极少主动感知

  • 100 Continue：服务器已接收请求头，客户端可继续发送请求体（如大文件上传，先确认服务器是否接受，再传数据）
  • 101 Switching Protocols：服务器同意切换协议（如从HTTP切换到WebSocket，用于实时通信场景）

2、 2xx 成功响应

  • 200 OK：请求完全正常，服务器返回预期数据，如打开网页、调用接口获取列表成功等
  • 201 Created：请求成功且“创建了新资源”，如POST接口创建用户、上传文件成功，服务器返回新资源的ID或地址
  • 204 No Content：请求成功，但服务器“无数据返回”，通常用于DELETE请求或不需要返回内容的操作
  • 206 Partial Content：客户端请求“部分资源”，如断点续传，视频分片加载，服务器仅返回请求的部分数据

3、 3xx 重定向响应

请求的资源不在原位置，客户端需根据服务器提示的新地址重新请求，分为“临时重定向”和“永久重定向”

  • 301 Moved Permanently：永久重定向，资源“永久迁移”到新地址，搜索引擎会更新索引，旧地址实效，需改用新地址
  • 302 Found：临时重定向，资源“临时在新地址”，旧地址仍可能恢复使用，所搜引擎不更新索引
  • 304 Not Modified：协商缓存命中，资源“未修改”，服务器无需返回完整数据，直接告知客户端“用户本地缓存即可”，优化加载速度，减少宽带消耗
  • 307 Temporary Redirect：临时重定向，与302相似，但更严格，要求客户端“必须使用原请求方法重新请求”，如原请求是POST，重定向后仍用POST，而302可能被改为GET

4、 4xx 客户端错误

“客户端的请求存在问题”，如地址错，权限不够、参数无效，服务器无法处理，需客户端修正后重新请求

  • 400 Bad Request：请求参数错误或格式无效，服务器无法理解请求内容
  • 401 Unauthorized：未授权，请求需要身份验证，如登陆，但客户端未提供或提供的凭证无效
  • 403 Forbidden：禁止访问，客户端已通过身份验证，但没有权限访问该资源，服务器明确拒绝，即使登陆也无法访问
  • 404 Not Found：请求的资源不存在，如网址错误，页面被删除
  • 405 Method Not Allowed：请求方法不支持，服务器仅支持特定的方法访问该资源，如只允许GET，客户端用了POST
  • 408 Request Timeout：客户端请求超时，服务器等待太久未收到完整请求，如网络卡顿，请求发送一半中断
  • 413 Payload Too Large：请求体过大，如上传的文件超过服务器限制的大小
  • 429 Too Many Requests：客户端请求过于频繁，触发服务器的限流策略，如接口限制每分钟最多调用10次，客户端1分钟内调用了20次

5、 5xx 服务器错误

客户端请求本身没问题，但服务器在处理过程中出现故障，需服务器排查修复

  • 500 Internal Server Error：服务器内部未知故障，如代码报错，数据库连接失败，内存溢出
  • 502 Bad Gateway：网关错误，服务器作为网关/代理时，无法从上游服务器，如后端服务，数据库，获取有效响应，如上游服务崩溃
  • 503 Service Unavailable：服务器暂时不可用，如服务器维护，过载停机，通常会告知恢复时间
  • 504 Gateway Timeout：网关超市，服务器作为网关时，等待上游服务器响应超时，如上游服务处理太慢，超过网关的等待时间
  • 505 HTTP Version Not Supported：服务器不支持客户端使用的HTTP版本，如客户端用HTTP/3.0，服务器仅支持HTTP/1.1
